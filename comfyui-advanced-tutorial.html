ComfyUI Architecture Workflows: From Beginner to Production

ComfyUI is the most powerful AI rendering tool available to architects right now. It is also the most misunderstood. This is the workflow guide we wish had existed when we started — no Python experience required.

ArchiGen AI Editorial
14 min read · Published April 2026 · No sponsored placements

Most architects who try ComfyUI abandon it within a week. The interface looks like a circuit diagram. The terminology — samplers, schedulers, CFG scale, denoise strength — reads like a physics textbook. There is no obvious start button. So people give up and go back to Midjourney, which has a Discord and a typing box and works immediately.

That is an understandable choice. It is also the wrong one if you care about the quality of your output. ComfyUI is not for everyone. But if you are doing serious architectural visualization work — anything that needs geometric fidelity, multi-pass consistency, or integration with your existing model outputs — then nothing else comes close.

This guide assumes you have no prior experience with ComfyUI. We will go from installation to a production exterior workflow in one read.

Why ComfyUI, and Why Now

The short answer: control. Every other AI image tool is a black box. You put a prompt in, an image comes out, and you have limited ability to influence what happens in between. ComfyUI exposes the entire pipeline as a node graph. Every step — the model, the conditioning, the sampler, the upscale method — is visible, editable, and saveable.

For architects, this matters for three reasons. First, repeatability. With ComfyUI you can save the exact workflow that produced your best render and reproduce it on a new project or a different angle. Second, image conditioning. ComfyUI's ControlNet integration lets you feed your own geometry — a SketchUp viewport, a Rhino line drawing, a depth map — and have the model respect it. No other browser-based tool does this as reliably. Third, cost. Once the software is installed locally, generation is free. A rendering practice that produces hundreds of iterations per week can save thousands per year compared to credit-based cloud tools.

Installation Without the Headache

What you actually need

You need a GPU with at least 8GB VRAM to run most SDXL models comfortably. An NVIDIA RTX 3070 or better handles the workflows in this guide without issues. AMD GPUs work but require additional setup steps (DirectML on Windows, ROCm on Linux). Apple Silicon M-series chips work via the MPS backend and run reasonably well on 16GB unified memory.

If you do not have a capable GPU, fal.ai offers ComfyUI workflows as an API — you design the graph locally, send it to their cloud, and get results back. It is slightly slower and costs credits, but it sidesteps the hardware requirement entirely. For architects at small firms without workstation-grade machines, this is a legitimate production path.

The install process

Download ComfyUI from the official GitHub releases page. Windows users: grab the portable package — it includes Python and all dependencies. Unzip, double-click run_nvidia_gpu.bat, and a browser tab opens at 127.0.0.1:8188. That is it for basic installation. No terminal, no pip commands, no environment management.

You will still need to download models separately. For architecture work, we use two base models: Juggernaut XL for photorealistic exteriors and RealVisXL when we want warmer, more atmospheric outputs. Both are free from Civitai. Drop the .safetensors file into your ComfyUI/models/checkpoints/ folder and it appears in the interface immediately.

Understanding the Node Graph

The ComfyUI interface is a canvas where processing steps are represented as nodes and connected by wires. Data flows from left to right. A minimal workflow has five nodes: a checkpoint loader (selects your model), a CLIP text encoder (processes your prompt), a KSampler (runs the diffusion), a VAE decoder (converts latent space back to pixels), and an image preview.

You do not need to understand the mathematics. What you need to understand is the flow: model → prompt → sample → decode → output. Everything else — ControlNet, upscalers, LoRAs, inpainting — slots into this chain.

Minimal ComfyUI Workflow — Node Chain
CheckpointLoaderSimple → CLIPTextEncode (positive)
CLIPTextEncode (negative) → KSampler
CheckpointLoaderSimple.VAE → VAEDecode
KSampler.LATENT → VAEDecode → PreviewImage

The Architecture-Specific Workflow: Image-to-Image with ControlNet

Pure text-to-image generation is useful for mood boards and early concept exploration. For anything that needs to respect your actual design geometry, you need image-to-image conditioning via ControlNet.

The workflow: export a viewport from SketchUp or Rhino as a clean line drawing or with the depth buffer (zbuffer/depth pass). Load this into ComfyUI as a conditioning image. The ControlNet preprocessor converts it into a format the model can read — a Canny edge map, a depth map, a normal map — and passes it alongside your text prompt into the sampler. The model then generates within the constraints of your geometry rather than inventing its own.

Which ControlNet preprocessor to use

Canny: Best for preserving hard edges — window lines, column grids, roof profiles. Use when geometric accuracy matters more than atmospheric softness. Works well for technical facades and orthographic views.

Depth (Midas or ZoeDepth): Best for preserving spatial relationships without locking every edge. Use for perspective views where you want the model to have some creative latitude on surface texture while keeping the massing correct. Our most-used preprocessor for client presentations.

Normal map: Best for interior work and surfaces with complex geometry. Less useful for typical exterior views.

Lineart: Good for hand-drawn sketch inputs — if you have a section perspective or a concept sketch and want to generate from it directly, this is the right choice.

ControlNetPreprocessor (type: depth_midas) →

The Multi-Pass Exterior Workflow

Single-pass generation is fine for quick ideation. For presentation-quality output, we run three passes.

Pass 1 — Massing and atmosphere (768×512, denoise 1.0)

This is your first generation from the SketchUp depth export. The goal is to establish the overall composition, light direction, and atmospheric feel. Do not obsess over material detail at this stage — the model will likely invent something plausible. Focus on getting the massing correct and the mood right. Try four to eight variations with different CFG values (6–10) and pick the strongest composition.

Prompt structure for this pass: lead with the overall scene type and lighting condition, then material keywords, then negative prompt. Example: "contemporary residential exterior, dusk, raked golden light from west, raw concrete base, dark timber cladding, minimalist landscaping, ultra-detailed architectural photograph". Negative: "cartoon, render, CGI watermark, people, cars, overexposed sky".

Pass 2 — Material refinement (img2img, denoise 0.4–0.6)

Take your best first-pass output into an image-to-image workflow. Drop the denoise strength to 0.4–0.6 so the model refines rather than regenerates. This is where you tighten material reads: push the concrete texture, sharpen the timber grain, add surface interest without breaking the composition. Small denoise adjustments have large effects — work in increments of 0.05.

If your first pass got the massing right but the materials look generic, this is where you fix it. If the composition itself is wrong, go back to Pass 1 rather than trying to fix it here. Trying to correct a fundamental composition problem in Pass 2 produces visible artifacts.

Pass 3 — Upscale and finishing (4× via tiled upscaler)

The standard approach: run the Pass 2 output through an Ultimate SD Upscale node set to 4× with a tile size of 512 and a denoise of 0.2–0.3. This adds genuine texture detail at high resolution without inventing geometry. Final output is typically 3072×2048 or similar — print-ready at A2.

If you want even sharper detail, pass the upscaled output through Leonardo.ai's Creative Upscaler. It adds a final layer of surface clarity that the ComfyUI upscaler alone rarely achieves. We use this on client-facing deliverables.

Four Tips That Save Hours

Save your seeds. Every KSampler node has a seed value that controls the random element of generation. When you find a composition you like, copy the seed number into a text file with the workflow parameters. Reproducing a specific output three months later from a seed is trivial. Trying to recreate it from visual memory alone is not.

Use workflow JSON files as a deliverable format. ComfyUI saves entire node graphs as JSON. We send these to clients alongside rendered images so they can see exactly which parameters were used. It is also your version control for rendering work.

The negative prompt is half your workflow. A carefully written negative prompt eliminates a category of bad outputs entirely. Our standard architecture negative: "people, vegetation front and center, cars, watermark, text, blur, low quality, cartoonish, unrealistic proportions, inverted colors, overly saturated, HDR halo, lens flare". Build your own over time based on the failures you keep seeing.

ControlNet strength is not a quality dial. Beginners turn it to 1.0 thinking more control means better results. What actually happens is the model becomes unable to synthesize convincing surface texture because the conditioning signal overwhelms the model's generative capacity. The sweet spot for exterior views from SketchUp depth maps is 0.65–0.80. Interior lineart can go higher, up to 0.90, because interior surfaces need to be more literal.

Models Worth Loading

Model choice matters more than most ComfyUI tutorials acknowledge. For architecture work, we have tested most of the major SDXL-based checkpoints and landed on three as our primary rotation.

Juggernaut XL v9 is our main workhorse for photorealistic exteriors. Excellent material rendering, good at architectural lighting conditions, handles concrete and glass particularly well. Free on Civitai.

RealVisXL v4 produces outputs with slightly more photographic warmth and less clinical precision. Better for residential projects where atmosphere matters more than technical accuracy. Also free.

FLUX.1 Dev (via GGUF quantized builds that run in ComfyUI) is the newest addition to our workflow. It handles complex prompt instructions better than SDXL-based models and produces results with exceptional material quality. Slower to generate, but the ceiling is noticeably higher. Run it through fal.ai if your local hardware is under spec.

When to Use ComfyUI vs. Everything Else

ComfyUI is the right tool when: you need geometric fidelity to an existing design, you are generating at volume and cannot afford per-image pricing, you need repeatability across multiple views or phases, or you want to build a custom multi-pass pipeline that goes beyond what any consumer tool offers.

It is not the right tool when: you need results in five minutes and have no prior workflow set up, you want atmospheric concept images without reference geometry, or you are sharing the work with someone who needs to operate the tool independently without training.

For that last case — quick concept atmospheres, client session iteration, fast moodboarding — Midjourney or Krea are faster and more accessible. ComfyUI and Midjourney are not competitors. They solve different parts of the workflow.

The question is never which tool is best. The question is which tool solves the specific problem in front of you.
